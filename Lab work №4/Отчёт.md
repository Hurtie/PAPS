## Документация по API

### 1. Концерты

#### 1.1. GET (Получения списка концертов)
Запрос на получение списка всех концертов  
**Входные данные**: Нет  
**Выходные данные**:  
Список концертов со следующей структурой
- id (string): Идентификатор концерта
- title (string): Название концерта
- date (timestamp): Дата концерта
- hall (string): Название зала
- created_at (timestamp): Время создания записи  

**Возможные ответы**:
- Успешный (200 OK)

#### 1.2. GET (Получение конкретного концерта по ID)
Запрос на получение конкретного концерта по его ID  
**Параметры пути**: id - Идентификатор концерта    
**Входные данные**: Нет

**Выходные данные**:
- id (string): Идентификатор концерта
- title (string): Название концерта
- date (timestamp): Дата концерта
- hall (string): Название зала
- created_at (timestamp): Время создания записи  

**Возможные ответы**:
- Успешный (200 OK)
- Концерт не найден (404 Not Found)

#### 1.3. POST (Создание концерта)
Запрос на создание нового концерта  
**Входные данные**:
- title (string): Название концерта
- date (timestamp): Дата концерта
- hall (string): Название зала  

**Выходные данные**:
- id (string): Идентификатор концерта   
- created_at (timestamp): Время создания записи  

**Возможные ответы**:
- Концерт создан (201 Created)
- Ошибка во входных данных (400 Bad Request)

#### 1.4. PUT (Обновление концерта)
Запрос на обновление данных концерта  
**Параметр пути**: id - Идентификатор концерта  
**Входные данные**:
- title (string): Название концерта
- date (timestamp): Дата концерта
- hall (string): Название зала  

**Выходные данные**:
- id (string): Идентификатор концерта   
- created_at (timestamp): Время создания записи
- updated_at (timestamp): Время обновления записи  

**Возможные ответы**:
- Концерт обновлён (200 OK)
- Концерт не найден (404 Not Found)
- Ошибка во входных данных (400 Bad Request)
#### 1.5. DELETE (Удаление концерта)
Запрос на удаление данных о концерте  
**Параметр пути**: id - Идентификатор концерта  
**Входные данные**: Нет  
**Выходные данные**: Нет  

**Возможные ответы**:
- Концерт удалён (204 No content)
- Концерт не найден (404 Not Found)

### 2. Билеты

#### 2.1. GET (Получение списка билетов на концерт)
Запрос на получение списка всех билетов на концерт
**Парамер запроса**: concert_id  
**Входные данные**: Нет  
**Выходные данные**:  
Список билетов со следующей структурой
- id (string): Идентификатор билета
- concert_id (string): Идентификатор концерта
- seat (string): Номер сидения
- price (string): Цена билета
- status (string): Доступность покупки
- created_at (timestamp): Дата создания записи  

**Возможные ответы**:
- Успешный (200 OK)

#### 2.2. GET (Получение конкретного билеты по ID)
Запрос на получение конкретного билета по его ID  
**Параметры пути**: id - Идентификатор билета  
**Входные данные**: Нет  

**Выходные данные**:
- id (string): Идентификатор билета
- concert_id (string): Идентификатор концерта
- seat (string): Номер сидения
- price (string): Цена билета
- status (string): Доступность покупки
- created_at (timestamp): Дата создания записи 

**Возможные ответы**:
- Успешный (200 OK)
- Билет не найден (404 Not Found)
#### 2.3. POST (Создание билета)
Запрос на создание нового билета, т.е. покупка  
**Входные данные**:
- concert_id (string): Идентификатор концерта
- seat (string): Номер сидения
- price (string): Цена билета 

**Выходные данные**:
- id (string): Идентификатор билета
- concert_id (string): Идентификатор концерта
- seat (string): Номер сидения
- price (string): Цена билета
- status (string): Доступность покупки
- created_at (timestamp): Дата создания записи  

**Возможные ответы**:
- Билет создан (201 Created)
- Ошибка во входных данных, место занято (400 Bad Request)
